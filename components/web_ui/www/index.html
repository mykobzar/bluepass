    ★ Recommended method — browser web flasher (no drivers needed)
    Open the Bluepass install page and click Install with flash encryption. The browser flashes the device directly over USB. No Python, no drivers, no command line.
    After flashing, the bootloader generates an AES-XTS key, burns it into eFuse, encrypts the flash, and reboots into Development mode automatically.
    Open web installer ↗
    Advanced: flash manually via esptool.py
    Download the -enc firmware variant from github.com/mykobzar/bluepass/tree/main/firmware and flash with esptool.py (Linux / macOS):
    esptool.py --chip esp32s3 --port /dev/ttyUSB0 --baud 460800 write_flash \ --flash_mode dio --flash_freq 80m --flash_size 4MB \ 0x0 bootloader-1.1.0-beta-enc.bin \ 0x8000 partition-table-1.1.0-beta-enc.bin \ 0x10000 ota_data_initial-1.1.0-beta-enc.bin \ 0x20000 bluepass-1.1.0-beta-enc.bin
    Replace /dev/ttyUSB0 with your actual serial port. On Windows use COM3 (or whichever COMx appears in Device Manager).
    ⚠︎ Irreversible action — read before proceeding
    Switching to Release mode permanently burns eFuse bits on this chip. This cannot be undone — not even by Espressif.

    You permanently lose:
    • UART flashing — idf.py flash will stop working
    • JTAG debugging — hardware debug access is closed forever
    • Encryption key — locked inside eFuse with no way to extract it

    What still works after switching:
    • OTA firmware updates via this web UI
    • Normal device operation

    Verify that OTA updates work correctly before proceeding. If OTA is broken and you switch to Release mode, the device cannot be reflashed.
