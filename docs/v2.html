Requires Chrome or Edge  ·  Web Serial API

Requires Chrome or Edge.
Download firmware for manual flashing with encryption.

⚠︎ Irreversible action — read before proceeding

Flashing this firmware will enable AES-XTS flash encryption on the device's first boot. The bootloader generates a unique key, stores it in eFuse, and encrypts the entire flash.


After encryption is enabled:

  • ✓  OTA updates via the web UI continue to work
  • ✓  All device features work normally
  • ✓  Settings tab shows Development mode — you can optionally lock to Release mode there
  • ×  Simple esptool.py write_flash no longer works (requires --encrypt)
  • ×  Encryption cannot be turned off — not even by Espressif
  • ×  If you lose OTA access and need to reflash, a computer with special tools is required

The encryption key is generated on-device and never leaves the chip. It is not recoverable if the device is damaged.

After flashing
1
Device boots and starts WiFi access point bluepass (open network)
2
Connect to it and open http://192.168.4.1
3
Enter your WiFi credentials → Save & Reboot
4
Short-press the button to activate the web UI on your local network
5
Go to Settings → Board to configure button and LED pins for your board
🔒
Encrypted install only: on first boot the device automatically enables AES-XTS encryption and reboots. This takes about 10 seconds — wait until the device restarts before connecting.
